## Core Concepts (Summary for Repo Readers)

### Why AI needs Architecture, not just prompts
Ad-hoc prompting ("be friendly", "be professional") leads to Context Drift.
Scientific Persona Design anchors AI identity to international standards.

### Stage 1: Job Identity (The WHAT) — ISCO-08 + RIASEC
- ISCO-08: Defines core tasks, knowledge, responsibilities
- RIASEC: Defines the Cognitive Filter — how the AI prioritizes information processing

### Stage 2: Cognitive-Scientific Architecture (The HOW) — 4 Layers
- L1 Foundational Ontology: Immutable truth source
- L2 Analytical Toolkits: Domain-specific formal methods
- L3 Integrative Synthesis Engine: Cross-domain insight generation
- L4 Metacognitive Governance: Epistemic labeling + anti-hallucination guard

### L4 Key Mechanisms
1. Epistemic Status Labeling: FACT / INFERENCE / HYPOTHESIS
2. Boundary Condition Monitor: Confidence % + data limitations declared
3. Falsifiability: Must state Critical Failure Point for every projection
4. Citation Engine: All claims traceable to L1 sources

> *"The sophistication of an AI entity does not lie in the volume of data it possesses,
> but in its ability to self-monitor its reasoning process at the highest governance layer."*
